Question 01: Discuss the scope of var, let, and const

Var:

Before the advent of ES6, var declarations ruled. There are issues associated with variables declared with var, though. That is why it was necessary for new ways to declare variables to emerge. First, let's get to understand var more before we discuss those issues.

Scope of var:

var declarations are globally scoped or function/locally scoped. The scope is global when a var variable is declared outside a function. This means that any variable that is declared with var outside a function block is available for use in the whole window. var is function scoped when it is declared within a function. This means that it is available and can be accessed only within that function.

Let:

let is now preferred for variable declaration. It's no surprise as it comes as an improvement to var declarations. It also solves the problem with var that we just covered. Let's consider why this is so..

Scope of Let:

A block is a chunk of code bounded by {}. A block lives in curly braces. Anything within curly braces is a block. So a variable declared in a block with let is only available for use within that block.

Const:

Variables declared with the const maintain constant values. const declarations share some similarities with let declarations.

Scope of Const:

Like let declarations, const declarations can only be accessed within the block they were declared. Const cannot be updated or re-declared. Question 02: Tell us the use cases of Null and Undefined

Undefined

Undefined is a primitive data type that represents a variable that has been declared but has not been assigned a value. It's important to note that `undefined` is not the same as "not-defined"it's a specific value that a variable can have.

Here are some common use cases for undefined:

  1. Variable Initialization:
    Variable Initialization: When you declare a variable without assigning an initial value to it, it is automatically initialized to `Undefined`.
  2. Function Parameters:
    If a function is called with fewer arguments than declared in its parameter list, the missing arguments are set to `Undefined`.
  3. Return Value
    If a function doesn't explicitly return a value, it implicitly returns `Undefined`.
  4. Accessing Non-Existent Object Properties:
    When you try to access a property of an object that doesn't exist, JavaScript returns `Undefined`.
  5. Array Elements:
    If you try to access an array element at an index that is out of range, it returns `Undefined`.
  6. Optional Function Parameters:
    When using optional function parameters, you can check if an argument is `Undefined` and provide a default value.
  7. Array Elements:
    Accessing an index that is out of bounds in an array will return `Undefined`.

Null

Null is a special value that represents the intentional absence of any object value or a placeholder for an expected object value that is not yet available.

It is often used in specific use cases,which given below:

  1. Initializing Variables:
    You can initialize a variable to null when you want to explicitly indicate that it has no value, but you expect it to hold an object value later in the program.
  2. Resetting or Clearing Values:
    Setting a variable to null is a common way to clear or reset it, especially when you want to release memory resources associated with an object.
  3. Function Return Values:
    Functions may return `null` to indicate that they didn't find a valid result or encountered an error.
  4. Object Properties:
    Object properties can be set to `null` to indicate that a certain property is intentionally empty or doesn't have a value yet.
  5. Checking for Missing Values:
    You can use null to check if a value is missing or uninitialized, especially when you want to distinguish between missing and defined values.
  6. Default Values:
    null can be used as a default value when you want to initialize variables or object properties and intend to set them to a different value later.

Question 03: What do you mean by REST API?

Api

An API for a website is code that allows two software programs to communicate with each other. The API spells out the proper way for a developer to write a program requesting services from an operating system or other application.

Rest Api

A Rest API is an architectural style for an application program interface (API) that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.

More About Rest Api

A REST API -- also referred to as a RESTful web service or REST API -- is based on representational state transfer (REST), which is an architectural style and approach to communications often used in web services development. REST technology is generally preferred over other similar technologies. This tends to be the case because REST uses less bandwidth, making it more suitable for efficient internet usage. RESTful APIs can also be built with programming languages such as JavaScript or Python.

How Rest Api works

A RESTful API breaks down a transaction to create a series of small modules. Each module addresses an underlying part of the transaction. This modularity provides developers with a lot of flexibility, but it can be challenging for developers to design their REST API from scratch. Currently, several companies provide models for developers to use; the models provided by Amazon S3, Cloud Data Management Interface (CDMI) and OpenStack Swift are the most popular.

REST API HTTP Method

A RESTful API uses existing HTTP methodologies defined by the RFC 2616 protocol, such as:
1.GET to retrieve a resource;
2.PUT to change the state of or update a resource, which can be an object, file or block;
3.POST to create that resource; and
4.DELETE to remove it.

REST API Data Formats Support

1. application/json
2. application/xml
3. application/x-wbe+xml
4. application/x-www-form-urlencoded
5. multipart/form-data
